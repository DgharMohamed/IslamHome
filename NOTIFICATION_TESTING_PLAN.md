## 🕌 1. إشعارات أوقات الصلاة (Prayer Notifications)

### 📂 الملفات المسؤولة:
- `lib/services/notification_service.dart` - الخدمة الرئيسية
- `lib/services/prayer_times_service.dart` - حساب أوقات الصلاة
- `lib/screens/settings_screen.dart` - واجهة الإعدادات

### 🎯 الوظائف المطلوب اختبارها:

#### 1.1 جدولة الإشعارات اليومية

**الوظيفة:** جدولة إشعارات الأذان لجميع الصلوات الخمس يومياً

**المتطلبات:**
- ✅ جدولة تلقائية عند تفعيل الإشعارات
- ✅ تحديث يومي للمواعيد
- ✅ دعم Exact Alarm على Android 12+

**كيفية الاختبار:**
```dart
// التحقق من الجدولة في الكود
await NotificationService().schedulePrayerNotifications();
```

#### 1.2 التذكير قبل الصلاة (15 دقيقة)

**الوظيفة:** إرسال تذكير قبل موعد الصلاة بـ 15 دقيقة

**المتطلبات:**
- ✅ إشعار منفصل قبل الأذان
- ✅ إمكانية تفعيل/تعطيل من الإعدادات
- ✅ نص واضح يوضح الوقت المتبقي

#### 1.3 تشغيل صوت الأذان

**الوظيفة:** تشغيل ملف صوتي للأذان عند موعد الصلاة

**المتطلبات:**
- ✅ تشغيل تلقائي عند الإشعار
- ✅ دعم ملفات MP3
- ✅ التحكم في مستوى الصوت

#### 1.4 صلاحيات Exact Alarm

**الوظيفة:** طلب صلاحية Exact Alarm على Android 12+

**المتطلبات:**
- ✅ فحص الصلاحية قبل الجدولة
- ✅ توجيه المستخدم للإعدادات
- ✅ رسالة توضيحية

#### 2.1 عرض معلومات الآية بالعربية

**الوظيفة:** عرض نص الآية والسورة بالعربية في الإشعار

- ⏩ التقديم 10 ثواني

```dart
// فحص المساحة قبل البدء
Future<bool> checkStorageSpace(int requiredBytes) async {
  final directory = await getApplicationDocumentsDirectory();
  final stat = await directory.stat();
  final freeSpace = stat.size;
  
  if (freeSpace < requiredBytes) {
    showDialog(
      context: context,
      builder: (context) => AlertDialog(
        title: Text('مساحة غير كافية'),
        content: Text(
          'المساحة المطلوبة: ${formatBytes(requiredBytes)}\n'
          'المساحة المتوفرة: ${formatBytes(freeSpace)}'
        ),
      ),
    );
    return false;
  }
  return true;
}

// إعادة المحاولة عند الفشل
Future<void> downloadWithRetry(String url, int maxRetries) async {
  int retries = 0;
  while (retries < maxRetries) {
    try {
      await download(url);
      return;
    } catch (e) {
      retries++;
      if (retries >= maxRetries) {
        throw Exception('فشل التحميل بعد $maxRetries محاولات');
      }
      await Future.delayed(Duration(seconds: 2 * retries));
    }
  }
}
```

### ❌ المشكلة 6: الإشعارات تختفي بعد إعادة التشغيل

**الأعراض:**
- الإشعارات المجدولة تختفي بعد إعادة تشغيل الجهاز
- يجب إعادة تفعيل الإشعارات يدوياً

**الأسباب المحتملة:**
1. ❌ عدم حفظ حالة الإشعارات
2. ❌ عدم الاستماع لحدث BOOT_COMPLETED

**الحلول:**


```dart
// في AndroidManifest.xml
<uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED"/>

<receiver android:name=".BootReceiver" android:enabled="true" android:exported="true">
  <intent-filter>
    <action android:name="android.intent.action.BOOT_COMPLETED"/>
  </intent-filter>
</receiver>

// في BootReceiver.kt
class BootReceiver : BroadcastReceiver() {
  override fun onReceive(context: Context, intent: Intent) {
    if (intent.action == Intent.ACTION_BOOT_COMPLETED) {
      // إعادة جدولة الإشعارات
      NotificationService.rescheduleNotifications(context)
    }
  }
}

// حفظ حالة الإشعارات
class NotificationService {
  Future<void> saveNotificationState() async {
    final prefs = await SharedPreferences.getInstance();
    await prefs.setBool('notifications_enabled', true);
    await prefs.setStringList('enabled_prayers', enabledPrayers);
  }
  
  Future<void> restoreNotificationState() async {
    final prefs = await SharedPreferences.getInstance();
    final enabled = prefs.getBool('notifications_enabled') ?? false;
    if (enabled) {
      await schedulePrayerNotifications();
    }
  }
}
```

- قلل من تكرار التحديثات للحفاظ على البطارية
- استخدم `setOngoing(true)` للإشعارات المستمرة
- نظف الإشعارات القديمة بانتظام

### 🔒 الأمان والخصوصية:

- لا تعرض معلومات حساسة في الإشعارات
- احترم إعدادات "عدم الإزعاج"
- اطلب الصلاحيات فقط عند الحاجة
- وضح للمستخدم سبب كل صلاحية
